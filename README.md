# 🏭 Mini TDD Factory

> **這是個許願池「只要你能把願望講清楚，AI 就能幫你實現。」**

不管你是什麼程度的開發者 (或 Vibe Coder)，這裡只有一個原則：**「把規格寫好，剩下的交給 AI。」**

---

## � 這是一個 GitHub Template (模板)

這不是一個給你直接 `git clone` 的專案，而是一個「工廠藍圖」。
你需要先把它複製一份，變成你自己的工廠。

### 第一步：建立你的工廠
1.  點擊本頁面上方的綠色按鈕 **"Use this template"** -> **"Create a new repository"**。
2.  幫你的新專案取個名字 (例如 `My-AI-App`)。
3.  把**你自己的新 Repo** Clone 到電腦上。

---

## 💡 它是怎麼運作的？ (自動開發)

這裡的運作方式叫 **TDD (測試驅動開發)**，在這個專案裡，我們把它自動化了 (The Magic Loop)。

### 1. 規格優先 (Spec-First)
開發軟體最大的痛苦不是寫程式，而是「改 A 壞 B」。
所以我們不寫程式，只寫「需求」：
1.  你在 `RFP` 資料夾寫下願望。
2.  AI 幫你寫出測試程式 (針對你的願望)。

### 2. 自動修正迴圈 (The Magic Loop)
啟動工廠後，它會進入一個 **「寫測試 -> 寫程式 -> 修正」** 的無限迴圈：
1.  **AI 嘗試寫程式**。
2.  **跑測試**：如果有紅燈 (錯誤)，AI 會看到錯誤訊息。
3.  **自動修正**：AI 會根據錯誤，自己修改程式碼，然後**再跑一次**。
4.  **直到綠燈**：直到所有測試都通過，它才會停下來跟你邀功。

> 簡單說：**它會自己 debug，直到做對為止。**

## 🧱 為了便於重構：樂高法 (Lego Method)

為了讓 AI 更好改程式，我們導入了「樂高化」的結構。
如果你選擇用它 (`LEGO_MODE=true`)，AI 會嚴格遵守以下原則，這對 AI 特別友善：

*   **壞了就換**：每個功能都是獨立的小積木。如果一個積木寫爛了，直接刪掉叫 AI 重寫一個，三秒鐘的事。
*   **一檔一事**：每個檔案只做一件事，絕對不超過 100 行。
*   **超級解耦**：A 積木壞了，絕對不會影響到 B 積木。

```text
/
├── contracts/  # � 說明書：定義資料長怎樣 (Schema)
├── tests/      # 🧪 測試儀：驗證積木是好的 (Tests)
├── actions/    # 🧱 積木區：核心邏輯，壞了直接丟掉重寫 (Logic)
└── entry/      # 🔌 接頭：API 或是 CLI 指令 (Entry Points)
```

## 🛠 開始製作你的第一個功能

### 1. 安裝環境
確保你有 Python 3，然後安裝必要套件：
```bash
pip install -r requirements.txt
```

### 2. 檢查設定
打開 `factory_config.md`，確認這兩行是對的：
```markdown
LANGUAGE=python
LEGO_MODE=true
```

### 3. 許願 (Make a Wish)
你只需要編輯這兩個檔案 (用中文寫)：
1.  **`RFP/requirements.md` (需求書)**：
    寫下你想做什麼。
    > *例：「我要做一個匯率轉換器，輸入台幣顯示美金...」*
2.  **`RFP/tasks.md` (任務單)**：
    把大目標切成小步驟，打勾勾給 AI 看。
    > - [ ] 取得目前匯率
    > - [ ] 實作轉換計算

### 4. 啟動工廠
在終端機輸入：
```bash
./factory.sh
```
然後你就可以去喝咖啡了。AI 會開始它的 Magic Loop，直到你看見綠燈。

## ⚙️ 進階設定 (factory_config.md)

除了樂高模式，我們還支援其他模式：

*   **`LEGO_MODE=true` (樂高模式)**：推薦。適合高品質、長久維護的專案。
*   **`LEGO_MODE=false` (標準模式)**：自由發揮，無架構限制，適合隨手寫的小腳本。
*   **`MODE=Dual` (Supervisor 模式)**：測試失敗時暫停，讓人類/IDE 介入 (適合搭配 Cursor/IDE)。
*   **`MODE=Single` (YOLO 模式)**：全自動掛機，失敗了 AI 會自己想辦法重試。
